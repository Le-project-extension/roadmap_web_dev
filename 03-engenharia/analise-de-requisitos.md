# Análise de Requisitos: Onde Tudo Começa

## Objetivo de Estudo
Sua missão neste módulo é entender que software não é feito de código, mas de soluções para problemas. Antes de abrir o editor de código, um engenheiro de software precisa dominar a arte de descobrir, entender e documentar o que precisa ser construído.

## Tópicos para Pesquisa: Conceitos Base
Pesquise e defina com suas próprias palavras os pilares da Engenharia de Requisitos:

1. **O que são Requisitos?** - Diferencie requisitos de **Funcionalidades**, **Restrições**, **Condições** e **Capacidades**.
   - Qual a diferença entre **Requisitos Funcionais** e **Requisitos Não-Funcionais**?

2. **Quem são os Stakeholders?**
   - Por que identificar todas as "partes interessadas" é vital para o sucesso do projeto? 
   - O que acontece se esquecermos um stakeholder importante no início?

3. **O que é Elicitação de Requisitos?**
   - Por que não chamamos apenas de "perguntar ao cliente"? 
   - Pesquise as principais **Técnicas de Elicitação** (Entrevistas, Questionários, Workshop, Brainstorming, Observação/Etnografia e Prototipação).

## O Ciclo de Vida dos Requisitos
A Engenharia de Requisitos segue um processo lógico. Pesquise o que deve ser feito em cada uma destas 7 etapas:

1. **Concepção:** Como o projeto nasce?
2. **Levantamento (Elicitação):** Como extrair a real necessidade do cliente?
3. **Elaboração:** Como refinar e modelar o que foi coletado?
4. **Negociação:** Como lidar com pedidos impossíveis ou conflitos de interesse?
5. **Especificação:** Como documentar para que o desenvolvedor entenda?
6. **Validação:** Como garantir que o que foi escrito é o que o cliente quer?
7. **Gestão de Requisitos:** Como lidar com as mudanças que surgirão durante o projeto?

## Missão Prática: O Investigador de Problemas
Para validar seu entendimento e garantir seus pontos, resolva o desafio abaixo:

**Cenário:** Um cliente pediu: *"Quero um sistema de vendas que seja seguro e aceite cartão."*

1. No seu repositório `missao-extensao-git`, crie uma Issue chamada `Análise de Requisitos: Sistema de Vendas`.
2. Identifique nesta frase: O que é um **Requisito Funcional**? O que é um **Requisito Não-Funcional**?
3. Liste pelo menos **3 Stakeholders** óbvios para este projeto.
4. Escolha **2 Técnicas de Elicitação** que você usaria para detalhar esse pedido e justifique por que as escolheu.
5. Escreva **3 perguntas** que você faria ao cliente para iniciar a fase de **Levantamento**.
