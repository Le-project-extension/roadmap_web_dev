# 📝 Markdown: A Linguagem da Documentação

## 🎯 Objetivo de Estudo
Sua missão neste módulo é entender como formatar textos na web de forma rápida e padronizada. O Markdown é o padrão absoluto na engenharia de software para escrever documentações, `READMEs`, comentários em Pull Requests e Issues.

## 🔍 Tópicos para Pesquisa
Antes de começar a escrever, garanta que você consegue responder:
1. O que é o Markdown e por que os desenvolvedores preferem usá-lo em vez de editores tradicionais como o Microsoft Word?
2. Como eu visualizo (renderizo) um arquivo `.md` no meu editor de código (como o VS Code)?

## ⌨️ Radar de Sintaxe
Pesquise como fazer as seguintes formatações usando apenas os caracteres do seu teclado. Você vai precisar disso para entregar os próximos desafios:

* Cabeçalhos (do Título principal H1 até o H6)
* Formatação de texto (Negrito, Itálico e Tachado/Riscado)
* Listas (Ordenadas numéricas e Desordenadas com *bullets*)
* Links e Imagens
* Citações (Blockquotes)
* Blocos de código (Inline para uma palavra e Multilinhas para blocos inteiros com destaque de sintaxe)
* Listas de tarefas (Checklists com caixas de seleção)

## 🚀 Missão Prática: O Perfil Dev
Esta etapa garante seus pontos de participação no projeto! Para provar que você domina a formatação básica, cumpra o desafio abaixo:

1. Abra o seu editor de código (como o VS Code).
2. Crie um arquivo chamado `meu-perfil.md`.
3. Escreva um mini-currículo seu utilizando **obrigatoriamente**:
   - Um título principal com seu nome.
   - Uma breve citação (Blockquote) sobre o que você espera do projeto de extensão.
   - Uma lista com 3 tecnologias que você já conhece ou quer aprender.
   - Um link clicável para o seu GitHub ou LinkedIn.
   - Um bloco de código contendo um simples `console.log("Hello World");` em JavaScript.
   - Um checklist com 2 tarefas (uma marcada como concluída e outra vazia).
