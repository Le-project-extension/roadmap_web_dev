# Scrum: O Motor da Agilidade

## Objetivo de Estudo
Sua missão neste módulo é entender como times de software profissionais se organizam para entregar valor de forma contínua, rápida e sem caos. Você vai descobrir como planejar o trabalho, dividir responsabilidades e inspecionar o que está sendo construído a cada ciclo.

## Tópicos para Pesquisa
Antes de falarmos de planejamento, garanta que você consegue responder:
1. O que é o **Manifesto Ágil** e quais são os seus 4 valores principais?
2. Por que o Scrum é definido como um *framework* e não como uma *metodologia* rígida?
3. Quais são os três pilares empíricos do Scrum (Transparência, Inspeção e Adaptação) e como eles se aplicam no dia a dia de um dev?
4. O que significa o termo *Timebox*?

## Radar de Conceitos
No Scrum, tudo se divide em Papéis, Eventos (Cerimônias) e Artefatos. Pesquise a função exata de cada um destes itens:

** Os Papéis (Quem faz o quê?):**
* Product Owner (PO)
* Scrum Master (SM)
* Developers (O Time de Desenvolvimento)

**Os Eventos (Quando fazemos?):**
* Sprint
* Sprint Planning (Planejamento)
* Daily Scrum (Reunião Diária)
* Sprint Review (Revisão)
* Sprint Retrospective (Retrospectiva)

**Os Artefatos (O que geramos?):**
* Product Backlog
* Sprint Backlog
* Incremento (Definition of Done - DoD)

## Missão Prática: O Primeiro Backlog
Vamos conectar o Scrum com o GitHub! Todo trabalho de um dev no Scrum começa com um item no Backlog. Para garantir seus pontos nesta etapa, você vai simular o planejamento de uma funcionalidade no seu próprio repositório.

1. Acesse o repositório `missao-extensao-git` que você criou no GitHub na missão anterior.
2. Vá até a aba **Issues** do seu repositório.
3. Imagine que você é o Product Owner e precisa pedir para o time criar uma **Tela de Login**.
4. Crie **3 Issues separadas**, quebrando essa funcionalidade em tarefas menores (ex: "Criar o HTML/CSS do formulário", "Validar se o email tem o formato correto", "Conectar botão de login com a API").
5. Use **Markdown** no corpo de cada Issue para descrever o que precisa ser feito de forma clara.
