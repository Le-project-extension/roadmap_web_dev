# Kanban: Gestao Visual de Fluxo

## Objetivo de Estudo
O foco deste modulo e compreender como gerenciar o trabalho de forma visual e continua. Voce deve investigar como o fluxo de tarefas impacta a produtividade da equipe e como identificar problemas no processo atraves da visualizacao.

![Ilustracao visual de um quadro Kanban: colunas de fluxo, cartoes e limites de WIP](https://artia.com/wp-content/uploads/2023/12/4.png)

## Temas para Pesquisa
Pesquise e defina os seguintes pontos fundamentais para a compreensao do framework:

1. **Origem do Kanban:** Como o sistema surgiu na industria automobilistica e como foi adaptado para a engenharia de software.
2. **Sistema Puxado vs. Sistema Empurrado:** Qual a diferenca fundamental entre esses dois modelos de trabalho e qual o Kanban utiliza?
3. **WIP (Work In Progress):** O que significa este conceito e por que estabelecer um limite e a regra central do Kanban?
4. **Gargalos (Bottlenecks):** Como a visualizacao do quadro permite identificar onde o fluxo de desenvolvimento esta travado?
5. **Metricas de Fluxo:** Pesquise as definicoes de Lead Time e Cycle Time.

## Estrutura de um Quadro Kanban
Um quadro de desenvolvimento profissional costuma seguir uma estrutura de colunas para representar o estado da tarefa. Utilize a tabela abaixo para guiar sua pesquisa sobre cada etapa:

| Coluna | Desafio de Pesquisa |
| :--- | :--- |
| Backlog | Qual a diferenca entre o Backlog do Produto e as tarefas que estao prontas para iniciar? |
| To Do (A Fazer) | Quem decide qual tarefa deve ser a proxima a entrar no fluxo de execucao? |
| In Progress (Fazendo) | O que acontece com a produtividade quando o limite de WIP desta coluna e ignorado? |
| Testing / Review | Qual a importancia de uma etapa de revisao antes da conclusao final? |
| Done (Concluido) | Quais criterios determinam que uma tarefa saiu do fluxo e entregou valor? |

## Materiais de Apoio
1. **Guia Kanban (Atlassian):** Referencia sobre os principios, praticas e diferencas para outros frameworks.
2. **Documentacao do GitHub Projects:** Instrucoes tecnicas de como configurar quadros e colunas dentro da plataforma.

---

## Missao Pratica: Estruturacao do Board
Para validar seu aprendizado, voce deve configurar o ambiente de execucao do seu projeto:

1. Acesse a aba Projects no seu repositorio pessoal do GitHub.
2. Crie um novo projeto utilizando o template de Board.
3. Configure as colunas exatamente conforme a tabela apresentada neste guia.
4. Pesquise e configure um limite de WIP para a coluna In Progress dentro do GitHub Projects.
5. Vincule as Issues criadas no modulo de Scrum a este novo quadro e organize-as na coluna inicial.
